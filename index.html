对JavaScript中函数的一些理解

函数的传入参数

虽然在函数定义时会规定传入的参数，但在实际使用时并不在意这个规定（个数和类型），++可以任意传入参数++。因为在ECMAScript中参数使用内部的==argument对象==（与数组类似）表示的。函数体通过argument对象来访问数组，获取参数。在调用时实际上传入了几个，都会被装进这个数组（EMCAScript中的数组的每一项都可以保存任何类型）。传入的第一个参数是argument[0]…,实际传入了几个，通过访问argument.length访问。

解释

  1. 定义时希望传入一个参数，但是实际传几个都行
  2. 实际输入几个参数，arguments.length就是几
  3. 若没有传入参数，对应的arguments[i]就是undefined,所以与之相加得NaN。

函数的传入参数

由于不想c++或java那样严格按照函数定义时的传参要求来调用函数，所以也就没有重载。因为无法通过参数判断，是否重载。若出现两个函数名一样的函数，则后一个覆盖前一个

函数声明、函数表达式

解释

  1. 这两个方法除了在函数声明提升时不一样（第二种会出错）其他都一样。
  2. 函数是Function对象，所以可以复制给变量
  3. 函数名就是一个变量，所以函数可以作为值来使用。就可以把一个函数作为参数传递给另一个函数。
  4. 访问函数的指针而不执行函数，则去掉函数名后的（），当参数传递时就是，传递指针。
  5. 也可以在一个函数中返回另一个函数（嵌套），此时带（）。

每个函数都包括两个方法apply()、call()

apply()示例

Function.apply(obj,args)

  1. 方法能接收两个参数
  2. obj：这个对象将代替Function类里this对象
  3. args：这个是数组，它将作为参数传给Function（args–>arguments）

解释

  1. 创建了一个Student类的实例，这个实例就是一个对象，将其赋值给student变量。
  2. this就指向了student
  3. arguments对象就是传入的参数组成的数组：[“qian”,”21”,”一年级”]
  4. Person.apply(this,arguments);里的this替代了Persion类里的this对象，也就是student替代了Persion里的this。那么就有了student.name, student.age。这样student对象就继承了Persion类里面的属性。

总结：传入的this对象替代一个类的this对象，简单的说就是，传入的this对象去得到另一个类的方法和属性。这里就是用student对象取得了Persion类的方法。

对JavaScript的面向对象的一些理解

在其他面型对象语言中，都有类的概念，通过类来创建多个对象，这些对象有相同的属性和方法（在类中规定的）。可以理解为这些对象是同一个类型的。

但是ECMAScript的面向对象与其他语言有所不同，因为在JavaScript中并没有类的概念。这里的对象可以想象成一个无序列表，就是一组名值对，值时数据或函数。那么如何创建对象呢？？

创建对象的几个方法

1. Object构造函数或对象字面量

顺便提一句：上面的this就是persion这个对象

==缺点：使用同一个接口创建很多对象，会产生大量重复代码==

2. 工厂模式

说明

用一个函数来封装以特定接口创建对象的细节。createPersion（）接受参数来构建Persion对象。每次调用该函数，都返回包含3个属性、1个方法的对象

==缺点：无法知道对象类型==

3.构造函数模式

可用构造函数来创建特定类型，像Object()和Array()这样的原生构造函数，也可以自定义创建。

说明

  1. 用new操作符来创建对象
  2. this指向新的对象
  3. 执行构造函数中的代码，为对象添加属性
  4. 返回新对象
  5. 惯例：构造函数以大写字母开头

==重点：构造函数也是函数==

  1. 用new操作符来调用就是构造函数
  2. 不用new操作符就是普通函数

==缺点：每次创建实例，就要重新创建方法sayName()。 p1和p2都有一个sayName()方法，但他们不是同一个Function实例。==

4. 原型模式

每创建一个函数，都会自动有一个prototype属性（原型）——通过调用构造函数创建的对象实例的原型。可以让所有对象实例共享原型对象所包含的属性和方法

说明

  1. Person构造函数、Person的原型属性、Person的实例之间的关系：

    1. Person的prototype属性：指向原型对象Person.prototype;
    2. person.protptype.constructor: 又指回Person

    3. Person的实例person1和person2的内部属性[[Prototype]]: 也指向Person.prototype

      （原型对象中除了包含内部属性constructor外，还有添加的name、age、job、sayName（））

  2. 在读取某对象的某属性时，进行搜索，先搜索对象实例本身，没找到继续找原型对象

    1. person1.sayName()时，先找实例person1，它自己并没有sayName(),于是继续找pereson1的原型，原型有，于是读取保存在原型中的sayName();

    2. 先搜索person1实例本身时就找的了name属性，于是不再搜索其原型

    3. 用delete操作符删除了person1.name后，再去找name属性，又只能在原型对象中找了

  3. 那怎么确定一个属性时存在于实例中，还是存在于原型中？用hasOwnProperty()(是自己的吗)
  4. in操作符——对象能够访问指定属性就返回true（只要有，不管是实例中还是原型中）

只有person1重写name属性时，它自己才有，hasOwnProperty()才返回true。

  1. 原型动态性–原型上的修改能立即在实例上反映出来

虽然在原型中添加方法之前就创建了实例，但这个实例让可以方法该方法。因为person.Hello()是，会搜索实例本身，没有就去搜索原型，就有了。
